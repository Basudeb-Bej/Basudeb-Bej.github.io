## 🛡️ How to Build a Cybersecurity Career Repository

A strong cybersecurity GitHub repository should prove **practical skills**, not just theory. Use this structure:

1. **`writeups/`**
   - CTF challenge writeups (Web, Crypto, Forensics, OSINT)
   - Vulnerability walkthroughs from labs like PortSwigger, TryHackMe, Hack The Box

2. **`projects/`**
   - Small security tools (port scanner, log analyzer, password strength checker)
   - Blue-team scripts (SIEM log parsing, IOC extraction, alert triage helpers)

3. **`labs/`**
   - Home-lab setup notes (Kali, Wireshark, Splunk, Security Onion, Wazuh)
   - Network diagrams and screenshots of safe lab environments

4. **`notes/`**
   - Concise notes for topics like OWASP Top 10, MITRE ATT&CK, networking, Linux hardening
   - Certification prep notes (Security+, CEH, eJPT, PNPT, etc.)

5. **`resume/`**
   - One-page security-focused resume
   - A `skills-matrix.md` mapping tools → evidence in your repo

### What each project should include

- `README.md` with problem statement, approach, tools, and results
- Screenshots or terminal output
- Ethical/legal disclaimer (only test in authorized environments)
- "What I learned" and "What I'd improve next"

### 30-60-90 day plan

- **30 days:** publish 3 writeups + 1 beginner tool
- **60 days:** add 2 intermediate labs + improve documentation
- **90 days:** ship 1 polished capstone project and pin top 6 repos

### Pro tips

- Keep commits regular (weekly activity matters)
- Prioritize quality over quantity
- Never upload real exploits against unauthorized targets or sensitive data
- Add a clear profile README and pin your best cybersecurity repositories
